# Working in this repository (humans & agents)

This project is built by its owner, **Harshit Wandhare**, with AI agents acting
strictly as tools under the owner's control. This file is the contract for any
agent (Claude, Codex, or otherwise) contributing here.

## Ownership & authorship

- The owner is the sole author and decision-maker. **Commits, PRs, and docs are
  authored as the owner.** Agents must not add themselves as authors, committers,
  or `Co-Authored-By` trailers, and must not insert "generated by AI" notices.
- Outward-facing or irreversible actions (pushing, publishing, sending messages,
  deleting data) require explicit owner approval.

## Quality bar (non-negotiable)

Before any commit, all of these must pass locally and in CI:

```bash
ruff check . && ruff format --check .
mypy src/
pytest tests/unit tests/integration        # coverage gate enforced
```

- Conventional Commits, humanized messages — nothing that reads as AI slop.
- `from __future__ import annotations` everywhere; type-only imports under
  `TYPE_CHECKING`; public APIs typed and documented.
- New behavior ships with tests. New optional backends degrade gracefully.
- No secrets in the repo. No new heavy dependency without a reason recorded in
  an ADR.

## Architecture & intent

Read these before making changes:

- [`docs/NORTH_STAR.md`](docs/NORTH_STAR.md) — vision, roadmap, infra decisions.
- [`docs/design/HLD.md`](docs/design/HLD.md) / [`LLD.md`](docs/design/LLD.md) — design.
- [`docs/research/competitive-analysis.md`](docs/research/competitive-analysis.md) — positioning.
- [`CONTRIBUTING.md`](CONTRIBUTING.md) — setup, commit, adapter-authoring.

## Token & context discipline

Prefer small, independently verifiable changes. Reuse existing code and tests
rather than regenerating them. Treat the gates above as the correctness oracle
instead of re-reading the whole tree. Record durable decisions so future
sessions don't re-derive them.
